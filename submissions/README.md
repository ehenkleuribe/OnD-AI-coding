# Submissions

This folder holds all challenge deliverables submitted by participants during the engagement.

**Each weekly subfolder has its own README** with submission instructions, naming conventions, and the NOTES.md template that captures your hardware and observations. The notes you leave here are what collectively build the [findings](../findings/README.md).

---

## How to Submit

1. **Fork** the repo (or work in your branch if you have push access)
2. Navigate to the correct week folder: `submissions/week-N/`
3. **Create a subfolder named after your GitHub username**: `submissions/week-N/your-username/`
4. Add your deliverable files and a completed `NOTES.md` (template in each week's README)
5. Open a **Pull Request** — title it `[Week N] your-username`

> Your PR is your submission. The review process is also how the group sees each other's work.

---

## Folder Structure

```
submissions/
  week-1/    ← Setup Challenge: Tetris via chat UI
  week-2/    ← Coding Experience: Enhanced Tetris via IDE
  week-3/    ← Automation: CLI / scripting tool
  week-4/    ← MVP: demo-able application
```

---

## Why NOTES.md Matters

Every submission includes a short `NOTES.md`. This isn't overhead — it's the raw material that feeds the [Hardware Assessment](../findings/HARDWARE-TIERS.md) and [Use-Case Matrix](../findings/USE-CASE-MATRIX.md) in `findings/`. The more honest and specific your notes, the more useful the collective output is for colleagues who weren't in this engagement.
